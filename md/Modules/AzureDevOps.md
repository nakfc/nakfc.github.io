# Azure Boards
> 团队协作敏捷工具

Work item process
* Agile(敏捷)

采用的是迭代式开发.

一、敏捷开发技术的适用范围:
1.项目团队的人数不能太多
2.项目经常发生变更
3.高风险的项目实施
4.开发人员可以参与决策
优势：
敏捷确实是项目进入实质开发迭代阶段，用户很快可以看到一个基线架构版的产品。敏捷注重市场快速反应能力，也即具体应对能力，客户前期满意度高。
